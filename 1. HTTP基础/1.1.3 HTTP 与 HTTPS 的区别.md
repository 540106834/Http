# **HTTP** 和 **HTTPS** 的区别
从本质、端口、传输方式、安全性和使用场景等角度。

## 1️ 基本概念

| 协议    | 全称                                 | 作用                                       |
| ----- | ---------------------------------- | ---------------------------------------- |
| HTTP  | HyperText Transfer Protocol        | 超文本传输协议，明文传输数据                           |
| HTTPS | HyperText Transfer Protocol Secure | 安全超文本传输协议，在 HTTP 基础上增加了 **加密层**（SSL/TLS） |

简单理解：

* **HTTP**：就像寄明信片，别人可能看到内容。
* **HTTPS**：就像寄密封信，信封加了锁，别人无法看到内容。

---

## 2️ HTTP 与 HTTPS 的主要区别

| 特性         | HTTP           | HTTPS                   |
| ---------- | -------------- | ----------------------- |
| **端口**     | 默认 80          | 默认 443                  |
| **加密方式**   | 无，加密依赖应用层      | 基于 **TLS/SSL** 加密数据     |
| **安全性**    | 数据明文传输，易被窃听、篡改 | 数据加密传输，保证机密性、完整性和认证     |
| **性能**     | 稍快，无握手过程       | 有额外的握手和加密解密开销，但现在优化后影响小 |
| **证书**     | 不需要            | 需要由 CA（证书颁发机构）签发证书      |
| **URL 前缀** | `http://`      | `https://`              |

---

## 3️ HTTPS 的实现原理（简单概述）

HTTPS 实质是在 HTTP 之上 **加了 TLS/SSL 层**：

1. **握手阶段**：

   * 客户端向服务器发送请求，协商加密算法。
   * 服务器返回证书，客户端验证证书。
   * 双方生成会话密钥（对称密钥）。
2. **数据传输阶段**：

   * 使用对称密钥加密 HTTP 数据。
   * 确保数据机密性和完整性。
3. **好处**：

   * 防止数据被中间人窃听。
   * 防止数据被篡改。
   * 确保服务器身份可信。

>  总结：HTTPS = HTTP + TLS/SSL 加密。

---

## 4️ 使用场景

* **HTTP**：

  * 公共信息浏览，例如新闻、博客。
  * 内网通信可以使用 HTTP（性能优先）。
* **HTTPS**：

  * 登录、支付、传输敏感数据的网站。
  * API 调用，保证通信安全。
  * 现代网站普遍强制使用 HTTPS。

---

一句话总结：

> **HTTP 是明文传输，安全性低；HTTPS 在 HTTP 上加了加密层（TLS/SSL），保证数据安全和服务器可信。**
