
#  一、HTTP 头部概念

* **HTTP 头部**存在于请求和响应中，用于携带**元信息**
* 作用：

  * 描述客户端或服务器信息（User-Agent、Server）
  * 控制请求或响应行为（缓存、编码、跨域）
  * 携带状态信息（Cookie、Authorization）
* 格式：

```
Header-Name: Header-Value
```

---

# 1️ 分类

HTTP 头部主要分为三类：

| 类别                        | 说明                 | 示例                                                       |
| ------------------------- | ------------------ | -------------------------------------------------------- |
| **通用头部（General Headers）** | 不特定于请求或响应，通用信息     | `Cache-Control`, `Connection`, `Date`                    |
| **请求头（Request Headers）**  | 客户端发给服务器，描述请求信息    | `User-Agent`, `Accept`, `Authorization`, `Cookie`        |
| **响应头（Response Headers）** | 服务器返回给客户端，描述响应信息   | `Server`, `Set-Cookie`, `Content-Type`, `Location`       |
| **实体头（Entity Headers）**   | 描述请求或响应的实体主体（Body） | `Content-Length`, `Content-Encoding`, `Content-Language` |

> 注意：实体头可能出现在请求或响应中，用于描述请求体或响应体。

---

# 2️ 常用请求头（Request Headers）

| Header            | 作用            | 示例                                   |
| ----------------- | ------------- | ------------------------------------ |
| `Host`            | 指定请求的目标主机     | `Host: www.example.com`              |
| `User-Agent`      | 客户端信息（浏览器/设备） | `User-Agent: Mozilla/5.0 ...`        |
| `Accept`          | 可接受的内容类型      | `Accept: text/html,application/json` |
| `Accept-Encoding` | 可接受的编码方式      | `gzip, deflate, br`                  |
| `Accept-Language` | 可接受语言         | `Accept-Language: zh-CN, en`         |
| `Authorization`   | 身份认证信息        | `Authorization: Bearer <token>`      |
| `Cookie`          | 携带客户端 Cookie  | `Cookie: sessionId=abc123`           |
| `Content-Type`    | 请求体格式         | `application/json`                   |
| `Content-Length`  | 请求体长度（字节）     | `Content-Length: 123`                |
| `Origin`          | 跨域请求来源        | `Origin: https://example.com`        |
| `Referer`         | 来源页面 URL      | `Referer: https://example.com/page`  |

---

# 3️ 常用响应头（Response Headers）

| Header                        | 作用           | 示例                                            |
| ----------------------------- | ------------ | --------------------------------------------- |
| `Server`                      | 服务器软件信息      | `Server: nginx/1.24.0`                        |
| `Date`                        | 响应时间         | `Date: Mon, 16 Nov 2025 09:00:00 GMT`         |
| `Content-Type`                | 响应体类型        | `Content-Type: application/json`              |
| `Content-Length`              | 响应体长度        | `Content-Length: 456`                         |
| `Content-Encoding`            | 响应体压缩方式      | `gzip, br`                                    |
| `Set-Cookie`                  | 设置客户端 Cookie | `Set-Cookie: sessionId=abc123; HttpOnly`      |
| `Cache-Control`               | 缓存策略         | `Cache-Control: no-cache, no-store`           |
| `Location`                    | 重定向 URL      | `Location: /login`                            |
| `Strict-Transport-Security`   | HSTS 安全策略    | `Strict-Transport-Security: max-age=31536000` |
| `Access-Control-Allow-Origin` | CORS 跨域策略    | `Access-Control-Allow-Origin: *`              |

---

# 4️ 常用通用头部（General Headers）

| Header          | 作用       | 示例                                    |
| --------------- | -------- | ------------------------------------- |
| `Connection`    | 控制连接是否保持 | `Connection: keep-alive`              |
| `Cache-Control` | 缓存策略     | `Cache-Control: max-age=3600`         |
| `Date`          | 消息发送时间   | `Date: Mon, 16 Nov 2025 09:00:00 GMT` |

---

# 5️ HTTP 头部实战价值

1. **接口调试**

   * 确认 Content-Type、Authorization、Cookie 是否正确
   * 判断浏览器发出的请求和服务端返回的状态

2. **性能优化**

   * Cache-Control、ETag、Last-Modified 提升缓存命中率
   * Content-Encoding 提升传输速度

3. **安全防护**

   * HTTPOnly、Secure Cookie
   * HSTS、CSP、X-Frame-Options

4. **跨域/移动端适配**

   * Access-Control-Allow-Origin / Access-Control-Allow-Headers
   * Accept-Language 国际化处理

# 6️ 补充说明

* HTTP 头部可以通过 **浏览器 DevTools → Network** 看到请求和响应的所有头部
* 可以通过 curl 测试头部：

```bash
curl -I https://www.example.com
curl -X POST -H "Content-Type: application/json" -d '{"name":"Tom"}' https://api.example.com/users
```

---

 总结：

> HTTP 头部是客户端和服务器之间的元信息通道，理解它可以帮助你调试接口、优化性能、保证安全、处理跨域请求，是前端、后端、运维、安全工程师的必备技能。
